🧱 Entendendo o Padrão Builder

Um exemplo simples e didático com Pessoa e Endereço

✅ Definição formal do padrão Builder (GOF)

Builder é um padrão de projeto criacional que separa a construção de um objeto complexo da sua representação, permitindo criar diferentes representações usando o mesmo processo de construção.

🤔 O que é Builder de forma simples?

É uma forma organizada de criar objetos passo a passo, especialmente quando o construtor ficaria muito grande ou confuso.

💡 Exemplo prático

Vamos construir objetos do tipo Pessoa, que possuem:

  • Nome
  • Idade
  • Endereço (rua + cidade)
Em vez de usar um construtor com muitos parâmetros, usamos um PersonBuilder para montar o objeto passo a passo.

🧪 Exemplo de Código

1. Classe Pessoa

public class Pessoa {
    private String nome;
    private int idade;
    private Endereco endereco;

    public Pessoa(String nome, int idade, Endereco endereco) {
        this.nome = nome;
        this.idade = idade;
        this.endereco = endereco;
    }

    @Override
    public String toString() {
        return nome + " (" + idade + " anos), Endereço: " + endereco;
    }
}

2. Builder para Pessoa

public class PersonBuilder {
    private String nome;
    private int idade;
    private Endereco endereco;

    public PersonBuilder setNome(String nome) {
        this.nome = nome;
        return this;
    }

    public PersonBuilder setIdade(int idade) {
        this.idade = idade;
        return this;
    }

    public PersonBuilder setEndereco(String rua, String cidade) {
        this.endereco = new Endereco(rua, cidade);
        return this;
    }

    public Pessoa build() {
        return new Pessoa(nome, idade, endereco);
    }
}

3. Código do Cliente

public class Main {
    public static void main(String[] args) {
        Pessoa pessoa = new PersonBuilder()
            .setNome("Carlos")
            .setIdade(40)
            .setEndereco("Av. Brasil", "São Paulo")
            .build();

        System.out.println("Pessoa criada via Builder:");
        System.out.println(pessoa);
    }
}

✅ Por que usar o padrão Builder?

  • 📦 Evita construtores com muitos parâmetros.
  • 🔍 Torna o código mais legível e fácil de manter.
  • 🔄 Permite reutilizar o processo de construção para objetos diferentes.

📌 Conclusão

O Builder é ideal quando você quer montar objetos complexos passo a passo, mantendo clareza e flexibilidade no código.
