🏭 Entendendo o Padrão Abstract Factory

Um exemplo simples com cartões de aniversário e natal

✅ Definição formal do padrão Abstract Factory (GOF)

Abstract Factory é um padrão de projeto criacional que fornece uma interface para criar famílias de objetos relacionados ou dependentes sem especificar suas classes concretas.

Ele permite produzir objetos que devem ser usados juntos, garantindo consistência entre os produtos e desacoplamento entre o código cliente e as implementações concretas.

🤔 O que é Abstract Factory de forma simples?

É um modelo (padrão) para criar objetos de um mesmo "grupo" sem dizer exatamente qual classe será usada.

💡 Imagine o seguinte...

Você está criando um sistema que gera cartões de aniversário e cartões de natal.

Esses cartões podem ser:

  • ✉️ Simples
  • 💎 De luxo
Você quer garantir que, se o usuário escolher o "estilo simples", ambos os cartões (aniversário e natal) venham no estilo simples.

E se escolher "luxo", ambos venham com estilo de luxo.

🏭 O que a Abstract Factory faz?

Ela cria uma "fábrica" que sabe montar a família de objetos (cartões) que combinam entre si.

Você terá duas fábricas:

  • Fábrica Simples: cria cartões simples.
  • Fábrica Luxo: cria cartões de luxo.

🧪 Exemplo de Código

1. Interface da Fábrica

public interface FabricaCartao {
    CartaoAniversario criarCartaoAniversario();
    CartaoNatal criarCartaoNatal();
}

2. Fábrica Simples

public class FabricaCartaoSimples implements FabricaCartao {
    public CartaoAniversario criarCartaoAniversario() {
        return new CartaoAniversarioSimples();
    }

    public CartaoNatal criarCartaoNatal() {
        return new CartaoNatalSimples();
    }
}

3. Código do Cliente (quem usa a fábrica)

public class Main {
    public static void main(String[] args) {
        FabricaCartao fabrica = new FabricaCartaoLuxo(); // ou Simples

        CartaoAniversario aniversario = fabrica.criarCartaoAniversario();
        CartaoNatal natal = fabrica.criarCartaoNatal();

        aniversario.exibirMensagem();
        natal.exibirMensagem();
    }
}

✅ Por que usar esse padrão?

  • 🔁 Você troca estilos (simples/luxo) sem mexer no código principal.
    No exemplo, o código cliente usa a interface FabricaCartao para criar cartões, sem se preocupar se está usando a FabricaCartaoSimples ou a FabricaCartaoLuxo. Para mudar o estilo dos cartões, basta trocar a fábrica concreta, sem alterar o código que consome os cartões.
  • 🧩 Todos os objetos criados combinam entre si (coerência).
    Como a fábrica cria uma família de objetos (ex: CartaoAniversarioSimples e CartaoNatalSimples juntos), você garante que os cartões usados juntos tenham o mesmo estilo visual e comportamental, evitando misturas indesejadas.
  • 🔒 Reduz o risco de misturar coisas diferentes (ex: cartão simples de aniversário com cartão natal luxo).
    Ao usar fábricas específicas para cada estilo, você evita que objetos incompatíveis sejam combinados, mantendo a consistência e a integridade do sistema.

📌 Conclusão

Abstract Factory é como um cardápio de combos.

Você escolhe "Simples" ou "Luxo", e recebe todos os itens do combo daquele tipo, sem se preocupar com os ingredientes de cada um.
