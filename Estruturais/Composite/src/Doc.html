🗂️ Entendendo o Padrão Composite

Exemplo com arquivos e pastas representando estrutura de um sistema de arquivos

✅ Definição formal do padrão Composite (GOF)

Composite é um padrão estrutural que permite tratar objetos individuais e composições desses objetos de forma uniforme, formando uma estrutura em árvore.

Ele permite que clientes manipulem objetos simples (folhas) e complexos (compostos) da mesma forma.

🤔 O que é Composite de forma simples?

É um padrão que organiza objetos em uma hierarquia árvore, onde cada objeto pode ser um único item ou um grupo de itens.

💡 Exemplo prático

Você quer representar a estrutura de diretórios e arquivos de um sistema operacional.

Uma pasta pode conter arquivos e outras pastas — cada um deve ter um método para mostrar sua estrutura.

Você deseja que tanto arquivos quanto pastas sejam tratados como "componentes" com uma interface comum.

📦 Estrutura do projeto


<PROJETO>
│
├── src/
│   ├── composite/
│   │   ├── Componente.java     ← Interface comum para arquivos e pastas
│   │   ├── Arquivo.java        ← Classe folha
│   │   └── Pasta.java          ← Classe composta que pode conter outros componentes
│   │
│   └── app/
│       └── Main.java           ← Código cliente que monta e exibe a estrutura

🧪 Exemplo de Código

1. Interface Componente

public interface Componente {
    void mostrar(int nivel);
}

2. Classe Arquivo (folha)

public class Arquivo implements Componente {
    private String nome;

    public Arquivo(String nome) {
        this.nome = nome;
    }

    public void mostrar(int nivel) {
        System.out.println("  ".repeat(nivel) + "- " + nome);
    }
}

3. Classe Pasta (composto)

public class Pasta implements Componente {
    private String nome;
    private List<Componente> filhos = new ArrayList<>();

    public Pasta(String nome) {
        this.nome = nome;
    }

    public void adicionar(Componente c) {
        filhos.add(c);
    }

    public void remover(Componente c) {
        filhos.remove(c);
    }

    public void mostrar(int nivel) {
        System.out.println("  ".repeat(nivel) + "+ " + nome);
        for (Componente filho : filhos) {
            filho.mostrar(nivel + 1);
        }
    }
}

4. Código Cliente

public class Main {
    public static void main(String[] args) {
        Componente arquivo1 = new Arquivo("arquivo1.txt");
        Componente arquivo2 = new Arquivo("arquivo2.txt");
        Componente arquivo3 = new Arquivo("foto.png");

        Pasta pastaDocumentos = new Pasta("Documentos");
        Pasta pastaImagens = new Pasta("Imagens");
        Pasta pastaRaiz = new Pasta("Raiz");

        pastaDocumentos.adicionar(arquivo1);
        pastaDocumentos.adicionar(arquivo2);
        pastaImagens.adicionar(arquivo3);

        pastaRaiz.adicionar(pastaDocumentos);
        pastaRaiz.adicionar(pastaImagens);

        pastaRaiz.mostrar(0);
    }
}

✅ Por que usar esse padrão?

  • 🌳 Permite tratar grupos e objetos individuais de forma uniforme.
  • 🔄 Facilita a construção de estruturas em árvore.
  • 🔧 Simplifica o código cliente, pois ele não precisa diferenciar folhas de compostos.

📌 Conclusão

Composite é ideal para representar estruturas hierárquicas, como sistemas de arquivos, menus, gráficos, entre outros.
