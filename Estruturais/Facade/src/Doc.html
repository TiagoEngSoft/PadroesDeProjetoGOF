🎬 Padrão de Projeto: Facade

Uma fachada simples para sistemas complexos 🎛️

✅ Definição formal do padrão Facade (GOF)

Facade é um padrão estrutural que fornece uma interface unificada para um conjunto de interfaces em um subsistema.
Ele define uma interface de alto nível que torna o subsistema mais fácil de usar.

🤔 O que é o Facade de forma simples?

Imagine um sistema de Home Theater com áudio, vídeo e internet. Ligar tudo individualmente pode ser trabalhoso.
Com o Facade, você cria uma interface simplificada que executa todos os passos para o usuário — com um único comando!

📦 Estrutura do Projeto

.
└── src
    ├── sistema
    │   ├── SistemaAudio.java
    │   ├── SistemaVideo.java
    │   └── SistemaRede.java
    │── Main.java
    |── HomeTheaterFacade.java

🧪 Código Exemplo

Classe do subsistema de áudio

public class SistemaAudio {
    public void ligarAudio() {
        System.out.println("Áudio ligado.");
    }

    public void desligarAudio() {
        System.out.println("Áudio desligado.");
    }
}

Classe do subsistema de vídeo

public class SistemaVideo {
    public void ligarVideo() {
        System.out.println("Vídeo ligado.");
    }

    public void desligarVideo() {
        System.out.println("Vídeo desligado.");
    }
}

Classe do subsistema de rede

public class SistemaRede {
    public void conectarInternet() {
        System.out.println("Internet conectada.");
    }

    public void desconectarInternet() {
        System.out.println("Internet desconectada.");
    }
}

Classe Facade

public class HomeTheaterFacade {
    private SistemaAudio audio;
    private SistemaVideo video;
    private SistemaRede rede;

    public HomeTheaterFacade() {
        audio = new SistemaAudio();
        video = new SistemaVideo();
        rede = new SistemaRede();
    }

    public void ligarSistema() {
        audio.ligarAudio();
        video.ligarVideo();
        rede.conectarInternet();
        System.out.println("Sistema de Home Theater pronto para uso!");
    }

    public void desligarSistema() {
        audio.desligarAudio();
        video.desligarVideo();
        rede.desconectarInternet();
        System.out.println("Sistema de Home Theater desligado.");
    }
}

Classe Main (Teste)

public class Main {
    public static void main(String[] args) {
        HomeTheaterFacade homeTheater = new HomeTheaterFacade();

        System.out.println("Ligando o sistema:");
        homeTheater.ligarSistema();

        System.out.println("\\nDesligando o sistema:");
        homeTheater.desligarSistema();
    }
}

🧠 Explicando o Código

  • 🔊 SistemaAudio, 📺 SistemaVideo, 🌐 SistemaRede: São partes do sistema que têm responsabilidades específicas.
  • 🎯 HomeTheaterFacade: É a fachada (facade) que encapsula a complexidade dos subsistemas e fornece uma interface simples ao cliente.
  • 🖥️ Main.java: O cliente que usa a fachada para interagir com o sistema inteiro sem saber dos detalhes internos.
  • 💡 Isso caracteriza o padrão Facade: uma interface unificada para esconder a complexidade de subsistemas.

✅ Benefícios do Facade

  • Simplifica o uso de sistemas complexos.
  • Reduz o acoplamento entre cliente e subsistemas.
  • Centraliza o controle das operações do sistema.
  • Facilita futuras mudanças e manutenções.

⚠️ Cuidados

  • ⚠️ O Facade não impede o uso direto das classes internas (ele apenas facilita).
  • 🧱 Pode se tornar muito grande se acumular muita responsabilidade.

📌 Conclusão

O padrão Facade oferece uma maneira elegante de esconder a complexidade de múltiplas classes colaboradoras, oferecendo um ponto de entrada único e simplificado.

Ideal para sistemas com muitos módulos ou bibliotecas que precisam de uma interface mais limpa para o cliente.
