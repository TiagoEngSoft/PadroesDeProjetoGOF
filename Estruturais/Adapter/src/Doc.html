🔌 Padrão de Projeto: Adapter

Um exemplo de Lobo em pele de Cordeiro e vice-versa 🐑🐺

✅ Definição formal do padrão Adapter (GOF)

Adapter é um padrão de projeto estrutural que permite que interfaces incompatíveis trabalhem juntas.
Ele converte a interface de uma classe em outra esperada pelos clientes, tornando classes com interfaces incompatíveis compatíveis entre si.

🤔 O que é o Adapter de forma simples?

Imagine que você tem um Lobo com métodos de lobo (uivar(), atacar()), mas quer usá-lo como se fosse um Cordeiro (balir()).

O Adapter cria uma "casca" (ou disfarce) ao redor do objeto, permitindo que ele seja usado com uma interface diferente da original.

💡 Analogias: Lobo e Cordeiro

  • Cordeiro: Interface com o método balir()
  • Lobo: Interface com os métodos uivar() e atacar()
  • Adapter: Um objeto que "finge" ser outro implementando sua interface

Exemplos:

  • Lobo em pele de Cordeiro: Um Lobo adaptado para se comportar como um Cordeiro
  • Cordeiro em pele de Lobo: Um Cordeiro adaptado para parecer um Lobo

🧪 Código Exemplo

Interface Cordeiro

public interface Cordeiro {
    void balir();
}

Interface Lobo

public interface Lobo {
    void uivar();
    void atacar();
}

Adapter: Lobo em pele de Cordeiro

public class LoboEmPeleDeCordeiro implements Cordeiro {
    private Lobo lobo;

    public LoboEmPeleDeCordeiro(Lobo lobo) {
        this.lobo = lobo;
    }

    public void balir() {
        System.out.println("😇 Meeee... (parece dócil, mas...)");
        lobo.uivar();
        lobo.atacar();
    }
}

Adapter: Cordeiro em pele de Lobo

public class CordeiroEmPeleDeLobo implements Lobo {
    private Cordeiro cordeiro;

    public CordeiroEmPeleDeLobo(Cordeiro cordeiro) {
        this.cordeiro = cordeiro;
    }

    public void uivar() {
        System.out.println("😅 Auuu... (mas é só um cordeiro tentando)");
    }

    public void atacar() {
        System.out.println("🐑 Tentando atacar... mas só sabe balir:");
        cordeiro.balir();
    }
}

Classe Main (Teste)

public class Main {
    public static void main(String[] args) {
        Lobo loboReal = new LoboReal();
        Cordeiro cordeiroReal = new CordeiroReal();

        Cordeiro disfarceLobo = new LoboEmPeleDeCordeiro(loboReal);
        Lobo disfarceCordeiro = new CordeiroEmPeleDeLobo(cordeiroReal);

        System.out.println("=== 🐺 Lobo em pele de cordeiro ===");
        disfarceLobo.balir();

        System.out.println("\\n=== 🐑 Cordeiro em pele de lobo ===");
        disfarceCordeiro.uivar();
        disfarceCordeiro.atacar();
    }
}

✅ Benefícios do Adapter

  • Permite reutilizar código existente com interfaces diferentes
  • Promove a compatibilidade entre objetos de APIs diferentes
  • Evita modificações em classes existentes

⚠️ Cuidados

  • ⚠️ Complexidade oculta: Muitos adapters em um sistema podem tornar o fluxo difícil de entender, criando camadas de abstração desnecessárias.
  • 🚫 Adapter mal utilizado pode mascarar problemas de design: Se você precisar adaptar muitas coisas o tempo todo, talvez a arquitetura precise ser repensada.
  • 🔄 Adaptação incompleta: Nem sempre é possível adaptar perfeitamente um objeto à nova interface, o que pode causar comportamentos inesperados.
  • 🔗 Acoplamento acidental: Apesar de ajudar no desacoplamento, adapters mal projetados podem criar dependências não intencionais entre classes.

📌 Conclusão

Adapter funciona como uma “máscara” que adapta o comportamento de um objeto para se encaixar no sistema.

No nosso exemplo, o Lobo pode se passar por Cordeiro (ou vice-versa), e isso nos ajuda a compreender como o padrão permite compatibilidade entre interfaces diferentes sem alterar suas implementações originais.
