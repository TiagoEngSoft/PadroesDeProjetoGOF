🔗 Padrão de Projeto: Chain of Responsibility

Delegando a responsabilidade de forma encadeada 💬🧑‍💻

✅ Definição formal (GoF)

O padrão Chain of Responsibility permite que vários objetos tenham a chance de processar uma solicitação, evitando o acoplamento entre o remetente e o receptor, ao encadear os objetos em uma cadeia.

🤔 O que é Chain of Responsibility na prática?

Imagine que um cliente abre um chamado de suporte.

- Primeiro, o Atendente tenta resolver.
- Se ele não conseguir, o chamado é encaminhado ao Suporte Técnico.
- Se ainda assim não for resolvido, vai para o Especialista.

Essa é a ideia do padrão: passar a responsabilidade ao próximo até que alguém trate.

🧪 Código Exemplo

Classe Chamado

public class Chamado {
    private String descricao;

    public Chamado(String descricao) {
        this.descricao = descricao;
    }

    public String getDescricao() {
        return descricao;
    }
}

Classe abstrata Responsavel

public abstract class Responsavel {
    protected Responsavel proximo;

    public void setProximo(Responsavel proximo) {
        this.proximo = proximo;
    }

    public abstract void tratarChamado(Chamado chamado);
}

Classe Atendente

public class Atendente extends Responsavel {
    public void tratarChamado(Chamado chamado) {
        if (chamado.getDescricao().toLowerCase().contains("senha")) {
            System.out.println("Atendente resolveu: " + chamado.getDescricao());
        } else if (proximo != null) {
            proximo.tratarChamado(chamado);
        }
    }
}

Classe SuporteTecnico

public class SuporteTecnico extends Responsavel {
    public void tratarChamado(Chamado chamado) {
        if (chamado.getDescricao().toLowerCase().contains("conectar")) {
            System.out.println("Suporte Técnico resolveu: " + chamado.getDescricao());
        } else if (proximo != null) {
            proximo.tratarChamado(chamado);
        }
    }
}

Classe Especialista

public class Especialista extends Responsavel {
    public void tratarChamado(Chamado chamado) {
        System.out.println("Especialista resolveu: " + chamado.getDescricao());
    }
}

Classe Main

public class Main {
    public static void main(String[] args) {
        Responsavel atendente = new Atendente();
        Responsavel suporte = new SuporteTecnico();
        Responsavel especialista = new Especialista();

        atendente.setProximo(suporte);
        suporte.setProximo(especialista);

        Chamado c1 = new Chamado("Esqueci minha senha");
        Chamado c2 = new Chamado("Não consigo conectar ao sistema");
        Chamado c3 = new Chamado("Erro crítico no banco de dados");

        atendente.tratarChamado(c1);
        atendente.tratarChamado(c2);
        atendente.tratarChamado(c3);
    }
}

🧠 Explicando o Código

  • Chamado: Representa uma solicitação do cliente, com uma descrição do problema.
  • Responsavel: Classe base que define o método tratarChamado() e referencia o próximo da cadeia.
  • Atendente: Resolve problemas simples como "senha". Se não resolver, encaminha.
  • Suporte Técnico: Resolve problemas técnicos como "conexão". Caso contrário, encaminha.
  • Especialista: Sempre resolve o que chegar, pois está no final da cadeia.
  • Main: Cria a cadeia e testa o tratamento de três chamados diferentes.

✅ Benefícios do Chain of Responsibility

  • 📦 Desacopla o remetente do receptor.
  • 🔀 Permite adicionar ou reorganizar os responsáveis com facilidade.
  • 🧩 Segue o princípio de responsabilidade única.
  • 🧵 Pode tornar o código mais limpo e extensível.

⚠️ Cuidados

  • ❗ Se ninguém tratar a requisição, ela pode ser ignorada silenciosamente.
  • 🔁 Longas cadeias podem afetar a performance ou dificultar o rastreamento.
  • 🧪 Pode ser difícil de testar em cenários complexos.

📌 Conclusão

O padrão Chain of Responsibility é ideal para situações onde uma ação pode ser tratada por vários responsáveis, um de cada vez.

No exemplo do suporte técnico, mostramos como delegar tarefas de forma elegante, sem acoplamento rígido entre o cliente e os tratadores.
