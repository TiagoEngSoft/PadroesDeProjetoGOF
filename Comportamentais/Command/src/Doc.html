🕹️ Padrão de Projeto: Command

Encapsulando comandos para dispositivos como TV e Luz

✅ Definição formal do padrão Command (GOF)

Command é um padrão comportamental que encapsula uma solicitação como um objeto, permitindo parametrizar clientes com diferentes requisições, enfileirar ou registrar solicitações e suportar operações que podem ser desfeitas.

🤔 O que é o Command de forma simples?

Imagine um controle remoto. Cada botão pode executar uma ação diferente, como ligar a TV ou apagar a luz.

Com o padrão Command, cada botão encapsula um "comando", e o controle remoto apenas executa o comando associado ao botão.

🧪 Código Resumido

Interface Comando

public interface Comando {
    void executar();
}

Classe LigarLuz

public class LigarLuz implements Comando {
    private Luz luz;
    public LigarLuz(Luz luz) {
        this.luz = luz;
    }
    public void executar() {
        luz.ligar();
    }
}

Classe ControleRemoto

public class ControleRemoto {
    private Comando comando;
    public void setComando(Comando comando) {
        this.comando = comando;
    }
    public void pressionarBotao() {
        comando.executar();
    }
}

Classe Main

public class Main {
    public static void main(String[] args) {
        ControleRemoto controle = new ControleRemoto();
        Luz luz = new Luz();
        Comando ligarLuz = new LigarLuz(luz);

        controle.setComando(ligarLuz);
        controle.pressionarBotao(); // 💡 Luz ligada
    }
}

🧠 Explicando o Código

  • Comando: Interface que define o método executar(), comum a todos os comandos.
  • LigarLuz / DesligarTV etc: Implementam a interface e executam ações específicas.
  • ControleRemoto: Armazena um comando e executa quando o botão é pressionado.
  • Main: Configura o controle com os comandos e executa.

🎯 Benefícios do Command

  • Encapsula ações como objetos reutilizáveis.
  • Facilita o histórico de ações e suporte a desfazer/refazer.
  • Permite alterar comandos dinamicamente em tempo de execução.
  • Separação clara entre quem solicita e quem executa.

⚠️ Cuidados

  • ⚠️ Pode aumentar o número de classes.
  • 🔧 Estrutura mais complexa para ações simples.

📌 Conclusão

O padrão Command é ideal quando você deseja encapsular ações e desacoplar o emissor do receptor.

No exemplo, cada botão do controle representa um comando, e o controle apenas executa o que foi configurado. Isso torna o sistema flexível e fácil de estender.
